# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in OnlyTrust A2A, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

Instead, please email us at **security@onlytrust.ai** with:

1. A description of the vulnerability
2. Steps to reproduce the issue
3. Potential impact assessment
4. Any suggested fixes (optional)

## Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial Assessment**: Within 5 business days
- **Fix & Disclosure**: Coordinated with reporter, typically within 30 days

## Supported Versions

| Version | Supported |
|---------|-----------|
| 0.1.x   | Yes       |

## Scope

This policy applies to the OnlyTrust A2A API server and its dependencies. Smart contract vulnerabilities should be reported separately with contract addresses and chain details.
